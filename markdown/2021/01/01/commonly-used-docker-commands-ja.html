Dockerコマンドたち

よく使うDockerコマンド一覧

Jan 1, 2021 • 1 min read

markdown

ここでは、よく使うDockerコマンド一。詳細については、コマンドライン・リファレンスなどでご確認ください。

Docker本体用コマンド

Docker本体の管理：docker system

停止中のコンテナやdanglingなイメージなど不要なリソースを全て削除します。

docker system prune

Dockerイメージ用コマンド

Dockerイメージの作成：docker build

現在のディレクトリにあるDockerfileという名前のDockerfileからfooというコンテナのイメージを作成します。

docker build -t foo .

Dockerイメージの一覧を表示：docker images

Dockerイメージの一覧を表示します。

docker images

Dockerイメージを削除：docker rmi

fooという名前のイメージを削除します。

docker rmi foo

danglingなイメージを全て削除します。

docker rmi $(docker images -f “dangling=true” -q)

Dockerコンテナ用コマンド

コンテナの作成・実行：docker run

ubuntuのイメージをもとにコンテナを起動して、現在の日時を表示させます。

docker run ubuntu /bin/date

nginxのイメージをもとに、コンテナをバックグラウンド（デーモン）で起動します。

docker run -d nginx

nginxのstableイメージをもとに、コンテナをバックグラウンドで起動します。stableなどタグ指定のないときにlatestイメージが利用されます。

docker run -d nginx:stable

Nginxのイメージをもとに、fooという名前のコンテナをバックグラウンドで起動します。

docker run -d --name foo nginx

Nginxのstableイメージをもとに、webという名前のコンテナをバックグラウンドで起動し、ホストPC localhostの8080ポート宛のアクセスを、コンテナの80ポートに転送します。

docker run -d -p 8080:80 --name web nginx:stable

稼働中のコンテナでコマンド実行：docker exec

webという名前の稼働中のコンテナにコマンドを実行してディレクトリ・ファイル一覧を表示します。

docker exec web /bin/ls

webという名前の稼働中のコンテナにログインします。

docker exec -it web /bin/bash

ログの取得：docker logs

webというコンテナ名のコンテナのログを取得します。

docker logs web

稼働中のコンテナ一覧を表示：docker ps

稼働中のコンテナの一覧を表示します。

docker ps

停止中のコンテナも含めて全てのコンテナの一覧を表示します。

docker ps -a

稼働中のコンテナを停止：docker stop

webという名前の稼働中のコンテナを停止します。

docker stop web

稼働中のコンテナを全て停止します。

docker stop $(docker ps -q)

停止中のコンテナを開始：docker start

webという名前のコンテナを再開します。

docker start web

稼働中のコンテナを再開

webという名前のコンテナを再開します。

docker restart web

停止中のコンテナを削除：docker rm

停止中のwebというコンテナを削除します。

docker rm web

停止中コンテナを全て削除します。

docker rm $(docker ps -aq)
